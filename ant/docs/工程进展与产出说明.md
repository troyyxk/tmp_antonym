# 工程进展与产出说明（截至 2026-02-25）

## 1. 项目目标与交付边界

本项目当前明确的交付目标是：**RAG 约束检索模块**，核心问题是 Dense Retrieval 在“否定词/反义约束”场景下容易召回语义相关但逻辑冲突的文档（Hard Negatives）。

当前交付边界：

1. 提供可运行的两阶段检索 PoC（召回 + 约束过滤/重排）。
2. 提供可切换的 Stage-1 检索后端（lexical / 本地 dense / OpenAI-compatible）。
3. 提供可复现实验脚本与可导出的评估报告（Recall@K、CCR、Latency）。

---

## 2. 已完成工作（做了啥）

### 2.1 两阶段检索主链路

已实现 `ConstraintAwareRetriever`，采用：

1. Stage-1 Topical Recall：先取相关候选。
2. Stage-2 Constraint Scoring：对候选执行约束一致性打分，支持 hard filter 和加权融合。

融合形式：

\[
score_{final} = w_t \cdot score_{topical} + w_c \cdot score_{constraint}
\]

并支持最小约束阈值过滤。

### 2.2 约束模块（PoC）

已实现 `RuleBasedConstraintScorer`，支持：

1. 否定约束：`not/without/no`、`不含/无` 等。
2. 包含约束：`with/include/contain`、`包含/包括` 等。
3. 数值约束：`under/below/at most`、`低于/最多`、`over/at least` 等。
4. 基础词形归一与否定上下文识别（如 `peanut-free`、`without peanuts`）。

### 2.3 Stage-1 后端可插拔

已支持三种后端：

1. `lexical`：轻量词法召回（离线可用，零外部依赖）。
2. `bge`：`sentence-transformers` 兼容 dense 召回（模型名或本地路径）。
3. `openai`：OpenAI-compatible Embedding API 召回。

### 2.4 本地模型路径适配

已验证 `/data/xingkun/local_model` 可直接用于 dense 后端（通过 `--bge-model /data/...`）。

同时修复了本地指令模型常见问题：

1. tokenizer 缺失 `pad_token` 时，自动回退到 `eos_token`，避免初始化失败。

### 2.5 Demo + Offline Eval + 报告导出

1. `demo_constraint_rag.py`：快速查看单 query 的 Baseline vs Constraint-aware。
2. `eval_retrieval_backends.py`：统一评估多后端。
3. 评估报告导出：自动生成 `CSV/JSON`，包括 `summary + per_query`。

## 4. 关键决策与原因（是因为啥）

1. **选择两阶段架构**：将“话题相关性”和“约束一致性”解耦，直接对齐问题本质，避免单向量空间纠缠。
2. **保留 lexical 后端**：保证在无网络、无 API key、无模型下载条件下仍可完整跑通链路。
3. **增加 dense 可插拔接口**：为后续替换真实生产召回器（BGE/OpenAI/本地模型）提供低改造成本路径。
4. **增加本地模型适配与 pad fallback**：优先适配你已有的 `/data/xingkun/local_model`，减少外部依赖。
5. **评估脚本支持“不可用后端自动跳过”**：提高实验稳定性，避免单一依赖失败导致整次评估中断。
6. **报告默认导出 CSV/JSON**：直接服务论文写作、表格粘贴、复现实验留档。

## 5. 当前验证结果与价值（为了啥）

### 5.1 lexical 后端（top-k=3）

1. Baseline：Recall@3 = `0.611`，CCR = `0.667`，Latency = `0.066 ms`
2. Constraint-aware：Recall@3 = `0.611`，CCR = `1.000`，Latency = `0.598 ms`

结论：在不损失召回的情况下，显著提升约束遵循率。

### 5.2 本地模型后端（`/data/xingkun/local_model/Llama-3.2-3B-Instruct`，CPU）

1. Baseline：Recall@3 = `0.833`，CCR = `0.667`，Latency = `177.19 ms`
2. Constraint-aware：Recall@3 = `1.000`，CCR = `1.000`，Latency = `171.30 ms`

结论：在本地 dense 召回上，约束模块同样有效，且可复用到真实模型路径。

## 6. 当前限制

1. Stage-2 仍是规则打分器，泛化能力受规则覆盖面限制。
2. 当前评估集规模较小（PoC 级），不能代表正式论文结论。
3. dense 后端使用通用指令模型做 embedding，语义检索表现可能不如专用 embedding 模型。
4. 目前还未接入真实向量库与在线服务链路（如 Faiss/Milvus/pgvector + API 服务）。

## 8. 快速复现命令

```bash
cd /home/xingkun/ant

# 1) Demo：本地模型后端
python3 scripts/demo_constraint_rag.py \
  --topical-backend bge \
  --bge-model /data/xingkun/local_model/Llama-3.2-3B-Instruct \
  --bge-device cpu \
  --query 'phones under $500' \
  --top-k 3

# 2) Offline Eval：并导出报告
python3 scripts/eval_retrieval_backends.py \
  --backends bge \
  --bge-model /data/xingkun/local_model/Llama-3.2-3B-Instruct \
  --bge-device cpu \
  --top-k 3 \
  --output-dir reports/paper_tables \
  --run-name local_bge_v1
```
